# EPIC-01: 發票處理 & 商品價格池

## 概述
用戶存入發票後，系統解析商品明細並匯入全平台商品價格資料庫。所有用戶的發票資料共同構成價格情報來源。

## 目標
- 準確解析發票商品明細
- 商品名稱正規化，建立統一商品 ID
- 所有用戶的購買價格匯入同一個商品價格池

## 驗收標準
- [ ] 發票存入後自動解析出商品名稱、數量、單價、通路、日期
- [ ] 同一商品不同寫法能正規化為同一個商品 ID
- [ ] 價格資料庫可查詢任一商品在各通路的歷史價格

---

## Stories

### STORY-01-01: 發票明細解析
**作為** 系統
**我想要** 解析用戶存入的發票，提取商品明細
**以便於** 建立結構化的消費記錄

**驗收標準：**
- [ ] 解析出每筆商品的：名稱、數量、單價、小計
- [ ] 解析出發票層級的：日期、通路來源
- [ ] MVP 階段支援 CSV/JSON 假資料匯入

**估計：** 3 points

---

### STORY-01-02: 商品名稱正規化
**作為** 系統
**我想要** 將不同寫法的同一商品統一為同一個商品 ID
**以便於** 跨用戶、跨通路的價格比對能正確配對

**驗收標準：**
- [ ] 同商品不同寫法合併（如「舒潔衛生紙100抽」vs「舒潔 衛生紙 100抽」）
- [ ] 同商品不同規格區分處理（100抽 vs 200抽，換算單位成本）
- [ ] 支援人工修正配對結果
- [ ] 正規化準確率 ≥ 90%

**估計：** 8 points

---

### STORY-01-03: 商品價格池建立
**作為** 系統
**我想要** 將所有用戶的購買價格匯入統一的商品價格資料庫
**以便於** 跨用戶比價與價格趨勢分析

**驗收標準：**
- [ ] 每筆價格記錄包含：商品ID、單價、通路、購買日期、（匿名）用戶ID
- [ ] 可查詢任一商品的：各通路最低價、歷史均價、價格趨勢
- [ ] 新發票存入時即時更新價格池
- [ ] 資料去識別化，保護用戶隱私

**估計：** 5 points

---

### STORY-01-04: 通路識別與管理
**作為** 系統
**我想要** 正確識別發票來源通路
**以便於** 按通路（好市多、全聯、7-11、家樂福）分類比價

**驗收標準：**
- [ ] 支援辨識好市多、全聯、7-11、家樂福四大通路
- [ ] 通路名稱正規化（統一編號 or 店名對應）
- [ ] 可擴充新增通路

**估計：** 2 points
