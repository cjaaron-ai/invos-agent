# EPIC-02: 覆購產品定義 & 比價引擎

## 概述
根據全平台用戶的購買行為定義覆購型產品，當用戶存入發票時，AI Agent 即時比對該商品在其他用戶的購買記錄中的價格低點，計算用戶多花了多少錢。

## 目標
- 以全平台數據定義覆購型產品
- 用戶存發票後即時比價
- 計算每筆購買相對於最低價可省的金額

## 驗收標準
- [ ] 系統能根據全平台數據自動標記覆購型產品
- [ ] 用戶存入發票後，覆購商品即時顯示比價結果
- [ ] 比價結果包含：用戶購買價、全平台最低價、價差金額

---

## Stories

### STORY-02-01: 全平台覆購產品辨識
**作為** AI Agent
**我想要** 分析全平台用戶的購買數據，辨識覆購型產品
**以便於** 知道哪些商品是大家常買、值得比價的

**驗收標準：**
- [ ] 被 N 位以上不同用戶購買的商品標記為覆購型（N 可配置，預設 ≥ 5）
- [ ] 覆購產品目錄自動更新（新商品達門檻自動加入）
- [ ] 覆購產品分類（日用品、食品、清潔用品等）
- [ ] 可查看全平台覆購產品排行榜

**估計：** 5 points

---

### STORY-02-02: 跨用戶價格比對引擎
**作為** AI Agent
**我想要** 在用戶存入發票時，比對該商品在其他用戶記錄中的價格
**以便於** 告訴用戶這次買貴了還是撿到便宜

**驗收標準：**
- [ ] 即時比對用戶購買價 vs 全平台同商品近期最低價
- [ ] 最低價資訊包含：價格、通路、購買日期
- [ ] 比對時考慮時間窗口（如近 30 天的價格才有參考性）
- [ ] 同規格比對，不同規格換算單位成本後比對

**估計：** 8 points

---

### STORY-02-03: 省錢金額計算
**作為** AI Agent
**我想要** 計算用戶這次購買相對於最低價多花了多少錢
**以便於** 用具體金額讓用戶感受到差距

**驗收標準：**
- [ ] 計算公式：(用戶單價 - 最低單價) × 購買數量 = 可省金額
- [ ] 單品計算 + 整張發票加總
- [ ] 已經是最低價的商品標記為「買到好價格 ✅」
- [ ] 支援百分比顯示（貴了 X%）

**估計：** 3 points

---

### STORY-02-04: 商品價格趨勢分析
**作為** AI Agent
**我想要** 分析覆購商品的價格趨勢
**以便於** 判斷目前是高點還是低點

**驗收標準：**
- [ ] 提供商品在各通路的價格趨勢（近 30/60/90 天）
- [ ] 標示歷史最低價、歷史均價
- [ ] 判斷目前價位（高於均價 / 接近均價 / 低於均價）

**估計：** 5 points
