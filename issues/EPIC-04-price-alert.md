# EPIC-04: 低價訂閱 & 推播通知

## 概述
用戶可針對覆購產品訂閱低價通知，支援三種門檻模式。當其他用戶存入發票出現更低價格時，AI Agent 即時推播通知訂閱者，引導回訪查看詳情。

## 目標
- 支援多種低價提醒門檻
- 其他用戶發票觸發即時通知
- 通知引導回訪，形成閉環

## 驗收標準
- [ ] 支援三種門檻模式設定
- [ ] 新發票觸發價格更新時，符合條件的訂閱者收到推播
- [ ] 通知可引導回訪查看商品詳情

---

## Stories

### STORY-04-01: 低價提醒訂閱設定
**作為** 用戶
**我想要** 對覆購產品設定低價提醒
**以便於** 出現好價格時第一時間知道

**驗收標準：**
- [ ] 支援三種門檻模式：
  - **百分比門檻**：比我上次買的低 X%（如 15%）
  - **固定目標價**：低於 $N 通知我
  - **歷史低價**：出現全平台歷史最低價時通知
- [ ] 可選擇監控的通路範圍（全部 or 指定通路）
- [ ] 可設定提醒有效期限（持續 / 30天 / 自訂）
- [ ] 支援批次設定（一次訂閱多個商品）

**估計：** 5 points

---

### STORY-04-02: 發票驅動的價格觸發引擎
**作為** 系統
**我想要** 當任一用戶存入發票時，檢查是否觸發其他用戶的低價提醒
**以便於** 用真實購買數據驅動通知，而非爬蟲

**驗收標準：**
- [ ] 新發票存入 → 提取覆購商品價格 → 比對所有訂閱者的門檻
- [ ] 價格符合門檻條件時加入通知佇列
- [ ] 同一商品同一價格只通知一次（去重）
- [ ] 觸發延遲 < 5 分鐘（從發票存入到通知發出）
- [ ] 避免單一異常低價誤觸發（可選：需 ≥ 2 位用戶確認同價位）

**估計：** 8 points

---

### STORY-04-03: App 推播通知
**作為** 用戶
**我想要** 收到即時的低價推播通知
**以便於** 把握購買時機

**驗收標準：**
- [ ] 推播內容包含：
  - 商品名稱
  - 最新低價 & 來源通路
  - 你上次購買價
  - 便宜了 X%
  - 按你的常買數量計算可省 $Y
- [ ] 點擊通知跳轉至商品詳情頁
- [ ] 支援靜音時段設定（如 23:00-08:00 不打擾）
- [ ] 通知分級：普通低價 vs 歷史新低（歷史新低加強提示）

**通知範例：**
```
🔔 舒潔衛生紙 比你便宜 33%！
你上次：$189/包（全聯）
現在有：$126/包（家樂福，用戶實購確認）

提醒設定：低於 15% 通知 ✅（實際低 33%）
你常買 3包，可省 $189
[查看詳情]
```

**估計：** 5 points

---

### STORY-04-04: 提醒管理面板
**作為** 用戶
**我想要** 管理所有已訂閱的低價提醒
**以便於** 查看、修改、暫停、刪除我的提醒

**驗收標準：**
- [ ] 列表顯示所有訂閱：商品、門檻模式、門檻值、狀態
- [ ] 支援修改門檻值與門檻模式
- [ ] 支援暫停 / 啟用 / 刪除訂閱
- [ ] 顯示已觸發的通知歷史紀錄（上次通知時間、價格）

**估計：** 3 points

---

### STORY-04-05: 回訪商品詳情頁
**作為** 用戶
**我想要** 點擊通知後看到完整的商品比價資訊
**以便於** 決定要不要去買

**驗收標準：**
- [ ] 顯示商品在各通路的目前價格（來自全平台用戶發票）
- [ ] 價格趨勢圖（近 30/60/90 天）
- [ ] 標示歷史最低價、歷史均價、我的上次購買價
- [ ] 用戶購買後可存入新發票 → 回到主流程閉環

**估計：** 5 points
