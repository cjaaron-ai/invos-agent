# Invos Agent — User Journey

## Phase 1️⃣ 發票明細匯入
用戶存入發票 → 系統解析商品明細 → 匯入全平台商品價格池
- 解析：商品 / 數量 / 單價 / 通路 / 日期
- 商品名稱正規化 → 統一商品 ID
- 所有用戶的資料匯入同一個商品價格池

## Phase 2️⃣ 全平台覆購產品定義
根據所有用戶的購買行為定義覆購型產品
- 被多位用戶重複購買 → 標記為覆購型
- 覆購產品目錄持續更新

## Phase 3️⃣ 即時比價分析
用戶存發票 → Agent 比對其他用戶的購買記錄中的價格低點
- 計算：(用戶單價 - 最低單價) × 數量 = 可省金額
- 整張發票所有覆購商品加總

## Phase 4️⃣ 省錢報告
逐項列出比價結果 + 加總可省金額 + 低價來源
- 引導用戶對買貴的商品設定低價通知

## Phase 5️⃣ 低價通知訂閱
三種門檻模式：
1. **百分比門檻** — 比我上次買的低 X%
2. **固定目標價** — 低於 $N 通知
3. **歷史低價** — 出現全平台歷史最低價時通知

## Phase 6️⃣ 推播通知
其他用戶存入發票 → 該商品出現更低價 → 觸發訂閱者推播
- 價格來源 = 真實用戶發票，非爬蟲
- 通知包含：便宜 X%、按常買數量可省 $Y

## Phase 7️⃣ 回訪閉環
點通知 → 商品詳情（各通路價格、趨勢圖）→ 購買後存發票 → 回到 Phase 3
